Chapter 1: Database Management System

Data and Information

Data refers to raw facts and figures without context. Information is processed data that is meaningful and useful for decision-making.

Features of Information

  • Accuracy
  • Timeliness
  • Completeness
  • Relevance
  • Accessibility

Database and Its Purpose

A database is an organized collection of data, typically stored electronically. Its main purpose is to manage, retrieve, and store data efficiently and securely.

Terminologies in Database

  • Table: A collection of related data organized in rows and columns.
  • Field: A column in a table that represents a data attribute.
  • Record: A row in a table containing related fields.
  • Tuple: Another term for a record.
  • Object: A real-world entity stored in a database with attributes and behaviors.
  • Keys: Unique identifiers for records (e.g., Primary Key, Foreign Key).

Data Dictionary

A data dictionary is a repository of metadata, storing information about the structure, constraints, and content of the database.

Database Management System (DBMS)

DBMS is software that interacts with end users and databases to capture and analyze data.

Objectives

  • Efficient data management
  • Minimize data redundancy
  • Ensure data integrity and security
  • Enable data sharing and multi-user environment

Advantages

  • Improved data sharing
  • Data security
  • Data consistency
  • Backup and recovery

Disadvantages

  • High cost
  • Complexity
  • Performance overhead

Types of Database Models

  • Hierarchical Model
  • Network Model
  • Relational Model
  • Entity-Relationship Model

Integrity Constraints and Types

  • Domain Constraint: Restricts the values of attributes.
  • Entity Integrity: Ensures primary key is not NULL.
  • Referential Integrity: Ensures foreign keys reference valid records.
  • Key Constraints: Enforces uniqueness of data.

Normalization

Normalization is the process of organizing data to reduce redundancy and improve data integrity.

Normal Forms

  • 1NF: Eliminate repeating groups.
  • 2NF: Remove partial dependency.
  • 3NF: Remove transitive dependency.

Advantages

  • Reduces data redundancy
  • Improves data integrity

Disadvantages

  • Complex queries
  • Slower performance

Centralized and Distributed Database

Centralized: Data is stored in a single location.
Distributed: Data is stored across multiple sites.

Advantages

  • Centralized: Easier to maintain, better data integrity
  • Distributed: Better availability and fault tolerance

Disadvantages

  • Centralized: Single point of failure
  • Distributed: Complex to manage

Comparison

Distributed databases offer scalability and fault tolerance, while centralized databases are simpler and cheaper to implement.

Database Security

Database security involves protecting the database from unauthorized access or malicious threats.

Challenges

  • SQL injection
  • Data breaches
  • Insider threats

Security Measures

  • User authentication
  • Encryption
  • Access controls

Roles of DBA

  • Database design
  • Performance tuning
  • Security management
  • Backup and recovery

Practical Topics

DDL and DML Languages

DDL (Data Definition Language) is used to define database structures. DML (Data Manipulation Language) is used to manipulate data within the database.
